#+title: spectreshell モジュール API (emacs-module 境界)

~libspectreshell.so~ が ~emacs_module_init~ で defalias する Elisp 関数と、
その戻り値の plist 形式を記録する。[[file:design.org][design.org]] の
「モジュール API (案)」を Phase 2 の実装に合わせて具体化したもの。
Phase 3 (spectreshell.el) 以降はこのドキュメントを参照する。

* 公開関数

| 関数 | 引数 | 戻り値 |
|------+------+--------|
| ~(spectreshell--create ROWS COLS)~ | ROWS, COLS: 1〜65535 の整数 | TERM (user-ptr, finalizer 付き) |
| ~(spectreshell--feed TERM BYTES)~ | TERM: user-ptr、BYTES: unibyte 文字列 | 更新情報 plist |
| ~(spectreshell--resize TERM ROWS COLS)~ | TERM、ROWS/COLS: 1〜65535 の整数 | 更新情報 plist (全行 dirty) |
| ~(spectreshell--encode-key TERM KEY MODIFIERS)~ | TERM、KEY、MODIFIERS | unibyte 文字列 or nil |
| ~(spectreshell--encode-paste TERM TEXT)~ | TERM、TEXT: 文字列 | unibyte 文字列 |
| ~(spectreshell--encode-mouse TERM BUTTON ACTION ROW COL MODIFIERS)~ | TERM、BUTTON、ACTION、ROW/COL: 整数、MODIFIERS | unibyte 文字列 or nil |
| ~(spectreshell--release TERM)~ | TERM | nil (副作用のみ) |

TERM は ~spectreshell--create~ が返す user-ptr。他のモジュールの
user-ptr や整数など、うちの Term でない値を渡すと
~wrong-type-argument~ が signal される。 ~spectreshell--release~ 済みの
TERM に ~--feed~ / ~--resize~ / ~--encode-key~ / ~--encode-paste~ /
~--encode-mouse~ を呼ぶと ~spectreshell-terminal-released~ (~error~ の
子コンディション) が signal される。 ~--release~ 自体は二重に呼んでも
安全 (2回目は no-op)。

ROWS/COLS に整数以外を渡すと Emacs 本体の型検査により
~wrong-type-argument~ が、範囲外 (0 以下または 65536 以上) だと
~args-out-of-range~ が signal される。

* ~--feed~ / ~--resize~ の戻り値 plist

#+begin_src elisp
(:dirty ((ROW TEXT SPANS) ...)
 :scrolled-off ((TEXT . SPANS) ...)
 :cursor (ROW . COL)
 :cursor-visible t-or-nil
 :responses "raw-bytes-or-omitted"
 :alt-screen entered-or-left-or-omitted
 :title "title-or-omitted")
#+end_src

- *~:dirty~*: 変更のあった行。ROW は 0-origin の整数、TEXT は行の内容
  (下記「文字列の扱い」参照)、SPANS は下記のスタイル区間リスト。
- *~:scrolled-off~*: 可視領域上端から流れ出て確定化すべき行。
  ~(TEXT . SPANS)~ の cons で、SPANS は TEXT の後ろに続く proper list
  ではなく「SPANS というリスト値そのもの」を cdr に持つ点に注意
  (~(cdr entry)~ で SPANS 全体が取れる)。
- *~:cursor~* / *~:cursor-visible~*: 常に含まれる。
- *~:responses~*: PTY に送り返すべき応答バイト列 (DA/DSR 等)。
  何もなければキーごと省略される。
- *~:alt-screen~*: このバッチで alternate screen への出入りが
  あった場合のみ ~entered~ / ~left~ シンボルを持つ。変化なしならキー
  ごと省略される。
- *~:title~*: OSC 0/2 でタイトルが変更された場合のみ含まれる。

** SPANS

#+begin_src elisp
((START END . STYLE-PLIST) ...)
#+end_src

START/END は TEXT 内のコードポイントオフセット (Emacs の文字列
インデックスと一致、バイトオフセットでもセル幅でもない)。装飾のない
区間は SPANS に現れない (「span が無い = 既定描画」)。STYLE-PLIST は
proper list なので ~(START END . STYLE-PLIST)~ は実質
~(START END :fg 1 :bold t ...)~ という平らなリストになる。

STYLE-PLIST のキーはすべて値が既定 (装飾なし) の場合キーごと省略する:

| キー | 値 |
|------+----|
| ~:fg~ / ~:bg~ | パレット index (0〜255 の整数) または ~"#rrggbb"~ 文字列 (24bit色) |
| ~:bold~ / ~:italic~ / ~:faint~ / ~:strikethrough~ / ~:inverse~ | ~t~ |
| ~:underline~ | ~t~ (single) / ~double~ / ~curly~ / ~dotted~ / ~dashed~ |
| ~:hyperlink~ | URI 文字列 (OSC 8) |

* ~--encode-key~ の KEY / MODIFIERS

- *KEY*: 印字可能な1文字は長さ1の文字列 (例 ~"a"~, ~"あ"~)。
  特殊キーはシンボル: ~up down left right home end prior next insert
  delete backspace tab return escape f1~ 〜 ~f12~ の各シンボル。
- *MODIFIERS*: シンボルのリスト。認識するのは ~ctrl~ ~alt~ ~shift~
  ~super~ の 4 つ。 ~alt~ は Emacs の meta に対応させる (呼び出し側で変換する)。
  未知のシンボルは無視される。
- DECCKM (カーソルキーモード) や kitty keyboard protocol、
  modifyOtherKeys など端末モードへの追従はモジュール内部の端末状態
  (TERM が保持) に従って自動的に行われる。
- KEY が空文字列/2文字以上の文字列、または未知のシンボルの場合は
  ~error~ を signal する。

戻り値は PTY に送る unibyte 文字列。何も送る必要がない場合 (例:
モディファイアキー単体など) は ~nil~ になる。

* ~--encode-paste~ の TEXT

TEXT は通常の (マルチバイト) Lisp 文字列。bracketed paste モードが
有効なら =ESC[200~...ESC[201~= で囲んで返す。無効なら中身をそのまま
返す。改行は端末に送る規約に合わせて ~\n~ → ~\r~ に正規化される
(ghostty-vt のペーストエンコーダの挙動)。戻り値は unibyte 文字列。

* ~--encode-mouse~ の BUTTON / ACTION / ROW / COL / MODIFIERS

ghostty-vt はマウスレポートのエンコーダを公開していないため、ghostty
本体 (~src/Surface.zig~ の ~mouseReport~) のボタン番号割り当てを
~--encode-key~ と同じ層 (モジュール内) に移植したもの。TERM が保持する
マウストラッキングモード (mode 9/1000/1002/1003) とフォーマット
(mode 1005/1006/1015、無指定なら x10) に自動的に追従する。

- *BUTTON*: ~nil~ (ボタンを押さない motion のみ)、整数 ~1~ / ~2~ / ~3~
  (それぞれ left/middle/right)、またはシンボル
  ~wheel-up~ / ~wheel-down~ / ~wheel-left~ / ~wheel-right~ のいずれか。
  それ以外の整数・シンボルは ~error~ を signal する。
- *ACTION*: シンボル ~press~ / ~release~ / ~motion~ のいずれか。
  それ以外は ~error~ を signal する。
- *ROW / COL*: 0-origin の整数の端末セル座標
  (~--encode-key~ の KEY と違い、印字可能文字ではなくクリック位置)。
  範囲外 (0 未満または 65536 以上) は ~args-out-of-range~ を signal する。
- *MODIFIERS*: ~--encode-key~ と同じシンボルのリストを受け取るが、
  ghostty のマウスレポートは ~ctrl~ / ~alt~ / ~shift~ しか表現できないため
  ~super~ は無視される (未知のシンボル同様、エラーにはしない)。

マウストラッキングが無効 (TERM がどの ~?100x~ モードも受け取っていない)
なら常に ~nil~ を返す。トラッキングが有効でも、現在のモードがその
BUTTON/ACTION の組み合わせを報告しない場合 (例: mode 1000 での motion、
mode 1002 でのボタンなし motion) も ~nil~ を返す。戻り値は PTY に送る
unibyte 文字列。

* 文字列の扱い (unibyte / multibyte)

- *~--feed~ の BYTES*: 呼び出し側は unibyte 文字列で渡すこと。PTY
  からの生バイト列をそのまま ~copy_string_contents~ で読み出す。
  ERT (~spectreshell-module-test.el~) で実証したとおり、
  ~unibyte-string~ で作った 0x80 以上を含むバイト列は
  ~copy_string_contents~ → Zig 側バッファ → (~--encode-paste~ 経由で)
  ~make_unibyte_string~ という往復で1バイトも変化せず戻ってくる。
  一方、そのバイト列を実際に ~--feed~ で VT パーサに通すと、単独の
  0x80〜0xFF は妥当な UTF-8 として解釈できないため ghostty-vt 側で
  置換文字相当の表示になる (これは端末エミュレーションとして妥当な
  挙動であり、モジュール境界のバグではない)。
- *~:dirty~ / ~:scrolled-off~ の TEXT*: 画面上の文字列なので常に
  UTF-8 の multibyte Lisp 文字列 (~make_string~) として返す。
- *~:responses~*: DA/DSR 等の制御応答は unibyte 文字列
  (~make_unibyte_string~) として返す。
- *~--encode-key~ / ~--encode-paste~ の戻り値*: PTY にそのまま書き込む
  前提のバイト列なので unibyte 文字列。

* エラー処理の実装メモ (実装者向け)

- Emacs 本体の env 関数 (~extract_integer~ / ~copy_string_contents~ 等)
  は型不一致のとき自前で non-local exit (~wrong-type-argument~ 等) を
  積む。Zig 側はこれを ~non_local_exit_check~ で検知したら追加の
  signal を呼ばずに即座にエラーとして伝播する (二重に signal すると
  未定義動作になるため)。
- TERM の型検証は ~type-of~ が ~user-ptr~ であることに加え、
  ~get_user_finalizer~ が本モジュールの finalizer 関数と一致するかで
  行う (user-ptr には Emacs 側の型タグ機構がないため)。
- ~--release~ は ~set_user_ptr~ で user-ptr の中身を ~NULL~ にすることで
  二重解放を安全にしている。GC 時の finalizer は data が ~NULL~ なら
  何もしない。
