* High

** TODO H-1: カーソル/マウスの COL 座標系不一致 (セル列 vs 文字オフセット)
対象: =spectreshell.el= (=spectreshell--row-col-pos=, =spectreshell--posn-terminal-coords=),
=docs/module-api.org=

モジュールの =:cursor= の COL は ghostty =render.cursor.active.x= 由来の
*セル列*、=encodeMouse= の COL も同じくセル列前提。一方 =row.zig= の
TEXT は spacer セルを読み飛ばした *コードポイント単位* で、Elisp 側は
COL をバッファ文字オフセットとして =(+ (point) col)= / =(- pt bol)= で
計算している。全角文字 1 つにつき 1 桁ずつカーソル追従とマウス報告が
ずれる (grapheme cluster では逆方向にもずれる)。

対応方針: Elisp 側で =move-to-column= / =current-column= を使い表示桁
(セル列) で変換する。=docs/module-api.org= に =:cursor= と encode-mouse
の ROW/COL の単位 (セル列) を明記する。CJK を含む行のカーソル・マウス
座標の ERT を追加する。

** TODO H-2: RenderState の行 dirty フラグを消費後にクリアしていない
対象: =src/core.zig= (=buildUpdate=)

ghostty の =RenderState.row_data= の dirty フラグは「ハンドリングした側
が false に戻す」契約 (ghostty render.zig の doc コメント) だが、
=buildUpdate= は読むだけで戻していない。一度 dirty になった行は以後
すべての feed で =:dirty= に再送され続ける (3x5 端末に "ab" を feed 後、
空文字列を feed しても dirty=3 のままであることを実験で確認済み)。
Elisp 側の =equal-including-properties= スキップがあるため表示は壊れな
いが、毎チャンク全 dirty 行のテキスト抽出・比較が走り性能が劣化する。

対応方針: 抽出ループで =row_dirty[y] = false= を書き戻す。「変更なし
feed は =:dirty= が空」の回帰テストを追加する。

** TODO H-3: emacs_module_init がランタイム/env のサイズ検証をしていない
対象: =src/main.zig=

=runtime->size= / =env->size= を検証せずに登録処理へ進むため、
=make_unibyte_string= (vtable オフセット 312) を持たない古い Emacs に
ロードすると、ロード自体は成功した後の =spectreshell--feed= 呼び出しで
env 構造体末尾を越えたゴミ関数ポインタを call して Emacs ごとクラッシュ
する。emacs-module.h ドキュメント推奨のサイズチェックで非 0 を返して
ロード失敗にすべき。

** TODO H-4: CI と nix の検証経路が存在しない
対象: =.github/workflows= (不存在), =flake.nix=, =Justfile=

- GitHub Actions 等の CI がなく、テスト実行はすべて手動前提。
- =flake.nix= に =checks= 出力がなく =nix flake check= が何も検証しない。
  パッケージにも =doCheck= がなく =zig build test= すら走らない。
- =just test= (Zig) / =test-el= / =nix-check= をまとめて回すレシピがない。

ghostty は API 非安定 (design.org 明記) のため、依存更新・コミットごと
の回帰を自動検出できないのが現状最大の運用リスク。

対応方針: flake に =checks= を追加し、=just check= レシピと GitHub
Actions ワークフローを整備する。

** TODO H-5: ロード時のトップレベル module-load 副作用
対象: =spectreshell.el=

トップレベルの =(spectreshell-ensure-module-loaded)= により、モジュール
未ビルド環境では =(require 'spectreshell)= 自体が error になり init の
ロードが止まる。autoload コマンド経由でも同様。package-lint 的にも
ロード時副作用は警告対象。

対応方針: トップレベル呼び出しをやめ、=spectreshell-start= (モジュール
関数を最初に必要とする入口) で =spectreshell-ensure-module-loaded= を
呼ぶ。

* Medium

** Elisp: 堅牢性

*** TODO M-1: 更新パスがナローイングの影響を受ける
対象: =spectreshell.el= (=spectreshell--apply-update= 一式)

=save-restriction= + =widen= がなく、ユーザーが =C-x n n= 中にプロセス
出力が届くと =point-max= を端末領域終端と誤認し、=--trim-rows= の
=delete-region= などが端末領域を破壊する。

*** TODO M-2: 更新パスが undo リストを無制限に蓄積する
対象: =spectreshell.el= (=spectreshell--apply-update=)

eshell バッファは undo 有効のため、feed ごとの delete/insert が undo
エントリとして際限なく積まれ、大量出力で肥大化する。ジョブ終了後に
=C-/= すると確定済みスクロールバックまで巻き戻る。更新中は
=buffer-undo-list= を束縛して記録を抑制すべき。

*** HOLD M-3: 同一バッファでの並行ジョブで端末領域が交錯する
対象: =spectreshell-eshell.el= (=--attach= / gather advice)

=spectreshell--current= / =spectreshell-eshell--process= が単一スロット
のため、=command &= 実行中に次のコマンドを打つと後発 attach が先発を
上書きし、双方が「marker〜point-max」を自領域とみなして削除・上書きし
合う。先発の detach は後発ジョブ実行中でも無条件に semi-char モードを
解除する。

保留理由: 端末領域を複数持つ設計変更 (領域終端マーカーの導入、
per-process の領域管理) が必要で、docs/design.org の「最後のパイプライン
ステージのみ」簡略化の見直しを含む。別途設計してから対応する。

*** TODO M-4: alt screen 中のプロセス死で primary 画面が失われる
対象: =spectreshell.el= (=spectreshell-finalize=)

alt screen 表示中にプロセスが異常死すると =alt-saved= (primary 画面の
スナップショット) が復元されずに破棄される。正常終了時 (?1049l 経由)
と同様に finalize 時も primary 画面を復元すべき。

*** TODO M-5: sentinel が無保護で eshell がハングし得る
対象: =spectreshell-eshell.el= (=spectreshell-eshell--sentinel=)

=spectreshell-eshell--detach= が signal すると =eshell-sentinel= が呼ば
れず、=eshell-process-list= の掃除・終了ステータス記録が行われないまま
eshell がコマンド実行中扱いでハングする。=unwind-protect= で保護する。

*** TODO M-6: filter の nil ガードと dead process への書き込みガードがない
対象: =spectreshell-eshell.el= (=--filter=, =--attach= の send-fn)

- detach 後にプロセスプロパティが nil の状態で filter が呼ばれると
  =(spectreshell-term nil)= の wrong-type-argument になる。
- send-fn は =:responses= をプロセス死亡後の最終 feed でも送ろうとして
  dead process へ書き込みエラーになり得る。=process-live-p= ガードを。

*** TODO M-7: 中クリック (mouse-2) が mouse-yank-primary にフォールスルー
対象: =spectreshell.el= (semi-char keymap)

=down-mouse-2= のみ束縛しているため、端末側マウストラッキングが無効の
とき release で生成される =mouse-2= クリックイベントがグローバルの
=mouse-yank-primary= に落ち、端末領域へ生バッファテキストが挿入されて
表示が壊れる (=spectreshell--encode-paste= 経路を通らない)。

*** TODO M-8: リサイズ契機が window-size-change-functions のみ
対象: =spectreshell-eshell.el= (=--attach=)

非表示中に attach したバッファを既存ウィンドウへ表示し直した場合など、
ウィンドウサイズ変化を伴わない表示替えでは端末サイズが追従しない。
=window-buffer-change-functions= も併用する。

*** TODO M-9: setTitle の OOM 時 dangling ポインタ

*** TODO M-10: DSR (CSI 6n) が DECOM (origin mode) を無視する
対象: =src/handler.zig= (=deviceStatus=)

ghostty 本体の stream_handler は origin mode 有効時にスクロール領域
top/left を差し引いた相対座標で応答するが、ここは常に絶対座標を返す。
xterm 仕様に反する。

UTF-8 多バイト分割の回帰テストを追加する。

** Elisp

*** TODO L-1: ドラッグ追跡ループが整数イベントを黙って捨てる
対象: =spectreshell.el= (=spectreshell--track-mouse-drag=)

ループ脱出時に =unread-command-events= へ戻すのがシンボル/リストのみで、
整数イベント (通常のキー入力) が消失する。release 判定もボタン一致を
確認しないため、mouse-1 ドラッグ中の =down-mouse-2= が元ボタンの
release として報告される。

*** HOLD L-2: キーマップの穴 (backtab / 修飾付き TAB・RET・DEL)
対象: =spectreshell.el= (semi-char keymap, =spectreshell--event-to-key=)

=backtab= (GUI の S-TAB)、=C-<tab>= / =M-RET= / =C-<backspace>= 等が
未束縛で Emacs 側コマンドに流れる。=spectreshell--event-to-key= 自体も
ascii 特殊キー判定が raw イベント完全一致のため、束縛を足すだけでは
=M-RET= が =return= + alt にならない。

保留理由: イベント正規化 (=--event-to-key=) の基本ケース分解を修飾付き
ascii 特殊キーまで広げる設計変更が必要。単体で意味のある改修として別途
対応する。

** Zig

*** TODO L-3: charEvent の unshifted_codepoint が shifted 値のまま
対象: =src/keymap.zig= (=charEvent=)

大文字 KEY に対して =unshifted_codepoint = cp= (シフト後) を設定して
いる。kitty/CSIu プロトコル時に shift ビットが脱落し、アプリが C-a と
C-S-a を区別できない。ASCII 大文字は =toLower= した値を入れる。

*** TODO L-4: MODIFIERS の循環リストで Emacs がハングする
対象: =src/module.zig= (=collectModifiers=, =collectMouseMods=)

循環リストを渡すと car/cdr の funcall ループが終わらず C-g も効かない。
ループ回数上限を導入する。

*** TODO L-5: UTF-8 マウス座標 (mode 1005) のサロゲート域でエラーになる
対象: =src/core.zig= (=writeUtf8Coord=)

座標 55255〜57342 で =utf8Encode= がサロゲートエラーになり encode-mouse
全体が Lisp エラーで落ちる。xterm の 1005 実装上限 (2015) も踏まえ、
エンコード不能座標は x10 の 222 超過と同様 null を返す。

*** TODO L-6: buildUpdate 末尾の errdefer が不完全
対象: =src/core.zig= (=buildUpdate=)

=responses= の dupe 後や =toOwnedSlice= 成功後に後続の =try= が失敗
すると、非 arena アロケータでは owned slice がリークする (モジュール
経由は arena のため顕在化しない)。errdefer を末尾まで通す。

*** TODO L-7: createImpl で make_user_ptr が signal した場合の Term リーク
対象: =src/module.zig= (=createImpl=)

finalizer 未登録のまま =Term= がリークする (Emacs 側 memory-full 時
のみの極低確率パス)。=pendingExit= を見て =term.deinit()= する。

*** TODO L-8: copyStringContents のサブスライス返却は free 不整合の罠
対象: =src/emacs.zig= (=copyStringContents=)

=alloc(len)= した領域の =buf[0..len-1]= を返すため、arena 以外の
アロケータで free すると長さ不一致になる。現状の呼び出し元はすべて
arena だが、契約をコメントで明文化するか全長確保に揃える。

*** TODO L-9: registerAll の失敗を無視して init が常に 0 を返す
対象: =src/main.zig=

=define-error= / =defalias= の funcall が signal しても成功扱いになり、
部分登録状態でロード成功になり得る。=pendingExit= を見て非 0 を返す。

*** TODO L-10: Timespec の Linux 64bit 決め打ちを compile error で明示する
対象: =src/emacs.zig=

i64/i64 決め打ちのため 32bit や他 OS ターゲットでは vtable オフセット
がずれて即クラッシュする。build.zig は任意ターゲットを許しているので、
対象外プラットフォームは =@compileError= で弾く。

* 性能・UX (設計判断を要する改善)

** HOLD P-1: point の強制移動でスクロールバック閲覧が実質不能
対象: =spectreshell.el= (=spectreshell--move-point=)

毎 update で point と全ウィンドウの window-point をカーソル位置へ移す
ため、emacs モードで履歴閲覧中も出力のたびに引き戻される。eat/term
同様「point がカーソル位置にあったときだけ追従」方式を検討する。

保留理由: 追従条件 (semi-char モード中は常に追従すべきか等) の仕様
決めが必要。

** HOLD P-2: font-lock 有効時に dirty 行スキップ最適化が無効化される
対象: =spectreshell.el= (=spectreshell--apply-dirty=)

=fontified= プロパティにより =equal-including-properties= が常に不一致
になり、全 dirty 行を毎回書き換える。比較前のプロパティ正規化か
jit-lock 制御を検討する。H-2 対応後は影響が大きく下がるため保留。

** HOLD P-3: Zig ホットパスの intern / funcall 回数
対象: =src/module.zig=

キーワードシンボルを feed ごとに =intern= し直している。
=make_global_ref= での初期化時キャッシュ、=symbol-name= 文字列比較の
=eq= 比較化で削減できる。実測で問題になってから対応する。

** HOLD P-4: ReadonlyHandler 委譲でクエリ系が無応答
対象: =src/handler.zig=

XTGETTCAP・kitty keyboard query (CSI ? u)・DECRQM が黙殺され、neovim
等の起動時タイムアウト待ちの原因になり得る。DA と同様に横取りして応答
する価値があるが、各クエリの正しい応答値の調査が必要なため保留。

** HOLD P-5: --pad-rows の O(n^2) と modification hooks
対象: =spectreshell.el=

ループ 1 周ごとに =count-lines= で全域を数え直している。必要行数を
一度計算して一括 insert にする。更新パスでの
=inhibit-modification-hooks= 束縛も検討 (他モードのフック互換性と
トレードオフ)。

* 検証済みで問題なしと確認した点 (記録)

- モジュール境界の関数名・アリティ・plist 形式は Elisp 宣言と
  =registerAll= で完全一致。
- user-ptr の finalizer を型タグに使う検証、=--release= の二重解放
  安全化 (set_user_ptr null で GC finalizer を no-op 化) は正しい。
- ペーストのエスケープ注入は ghostty =paste.encode= が ESC・C-c 等の
  危険バイトを bracketed paste の有無によらず空白置換しており安全。
  =core.zig= が mutable コピーを渡すのはこの sanitize を働かせる正しい
  呼び方。
- 依存ピン留め (build.zig.zon hash + flake.lock + コミット済み
  build.zig.zon.nix)、ライセンス同梱、README / マニュアルの記述精度
  (TERM フォールバック・キーバインド・検出パス) はコードと一致。
